# Maniac Garage 디자인 방향성 가이드

## 1. 문서 목적

이 문서는 Maniac Garage 웹서비스의 전체 디자인 방향, 모바일 UI 구조, 랜딩 페이지 구성, 공개 장비 페이지, 탐색/커뮤니티 화면, 어드민에서 관리 가능한 디자인 요소의 기준을 정리한다.

사용자가 참고로 제시한 디자인은 넓은 여백, 모바일 우선 레이아웃, 간결한 상단 네비게이션, 카드형 콘텐츠, 가로 스크롤 섹션, 섹션별 추천 콘텐츠, 심플한 메뉴 드로어가 특징이다. Maniac Garage는 이 구조적 장점을 참고하되, 그대로 따라 하지 않고 **모던한 갤러리 감성 + 힙한 장비/차고 문화 + 프리미엄 기록장** 느낌으로 재해석한다.

---

## 2. 디자인 한 줄 방향

**정돈된 갤러리처럼 깔끔하지만, 마니아의 차고처럼 개성과 장비감이 느껴지는 모바일 우선 웹서비스.**

키워드:

- Modern
- Clean
- Garage
- Archive
- Premium
- Hip
- Minimal
- Tactile
- Collectible

서비스가 줘야 하는 느낌:

- 내 장비를 올리고 싶다.
- 기록이 예쁘게 남는다.
- 중고 판매 링크로 보내도 신뢰가 생긴다.
- 너무 딱딱한 정비 앱이 아니라 감각적인 포트폴리오 같다.

---

## 3. 참고 디자인에서 가져올 요소

### 3.1 가져올 것

- 모바일 우선 화면 밀도
- 넓은 여백과 심플한 섹션 구성
- 상단에 명확한 네비게이션
- 검색창 중심 탐색 구조
- 가로 스크롤 카드 리스트
- 섹션별 제목/설명/더보기 링크
- 둥근 카드와 이미지 중심 구성
- 카테고리 리스트를 카드형으로 보여주는 방식
- 메뉴 드로어의 간결한 항목 구조

### 3.2 그대로 따라 하지 않을 것

- Figma 특유의 리소스 마켓 느낌
- 너무 흰 배경만 있는 무난한 SaaS 느낌
- 디자인 리소스 플랫폼 같은 카테고리 구성
- 단순한 카드 나열만 반복하는 구조
- 브랜드 개성이 약한 평면적 화면

### 3.3 Maniac Garage식 재해석

- 흰 배경 기반이지만, 차콜/그라파이트/메탈릭 포인트를 섞는다.
- 카드에는 장비 사진이 크게 보이게 한다.
- 타임라인, 부품 리스트, 정비 기록 같은 데이터가 예쁘게 보이게 한다.
- 카테고리는 단순 텍스트보다 장비 문화가 느껴지는 카드로 표현한다.
- 공개 장비 페이지는 SNS 프로필보다 “장비 포트폴리오”처럼 보이게 한다.

## 4. 전체 비주얼 톤

### 4.1 기본 분위기

- 깔끔한 화이트/오프화이트 배경
- 강한 검정 텍스트
- 얇은 라인
- 충분한 여백
- 둥근 카드
- 큰 장비 이미지
- 차고/메탈/툴박스 느낌의 포인트 컬러

### 4.2 피해야 할 분위기

- 너무 귀여운 앱 느낌
- 너무 자동차 정비소 같은 투박한 느낌
- 과한 네온/사이버펑크
- 중고거래 앱처럼 시끄러운 UI
- 배너가 너무 많은 쇼핑몰 느낌
- 템플릿 마켓을 그대로 따라 한 느낌

## 5. 컬러 방향

### 5.1 기본 팔레트

```txt
Background: #F7F6F2 또는 #FAFAF7
Surface: #FFFFFF
Text Primary: #171717
Text Secondary: #666666
Border: #E7E5DF
Muted Surface: #F0EFEB
Graphite: #252525
```

### 5.2 포인트 컬러 후보

브랜드의 힙한 감성을 위해 포인트 컬러는 너무 흔한 파랑보다 장비 문화와 어울리는 색을 사용한다.

후보:

```txt
Garage Orange: #FF6A2A
Racing Lime: #C7FF3D
Electric Blue: #287CFF
Oil Green: #2E4F3E
Metal Purple: #7C5CFF
```

초기 추천:

```txt
Primary Accent: Garage Orange #FF6A2A
Secondary Accent: Graphite #252525
Highlight: Racing Lime #C7FF3D
```

주의:

- 포인트 컬러는 버튼, 배지, 링크, 상태 강조에 제한적으로 사용한다.
- 전체 화면을 컬러풀하게 만들지 않는다.
- 장비 사진이 주인공이 되도록 UI 색은 절제한다.

## 6. 타이포그래피

### 6.1 방향

- 제목은 굵고 명확하게
- 본문은 읽기 쉽게
- 숫자/주행거리/비용/스펙은 데이터 카드처럼 정돈
- 한글 가독성을 최우선으로 한다.

### 6.2 폰트 후보

한국어 기준:

- Pretendard
- SUIT
- Noto Sans KR

영문/숫자 포인트:

- Inter
- Geist
- IBM Plex Sans

초기 추천:

```txt
Primary Font: Pretendard
Numeric/Data Font: Inter 또는 Geist
```

### 6.3 크기 체계 예시

```txt
Display: 40~56px
Page Title: 28~36px
Section Title: 20~24px
Card Title: 15~18px
Body: 14~16px
Caption: 12~13px
Tiny Meta: 11~12px
```

모바일에서는 지나치게 작은 글씨를 피하고, 섹션 제목과 카드 제목의 계층을 명확히 한다.

## 7. 레이아웃 원칙

### 7.1 모바일 우선

초기 사용자는 모바일에서 장비 사진을 올리고 공유 링크를 확인할 가능성이 높다. 모든 화면은 모바일 기준으로 먼저 설계한다.

모바일 기준:

```txt
Page Padding: 20px
Section Gap: 40~64px
Card Radius: 18~28px
Card Gap: 12~16px
Bottom Safe Area 고려
```

### 7.2 데스크톱 확장

데스크톱에서는 단순히 화면을 늘리지 말고 콘텐츠 폭을 제한한다.

```txt
Content Max Width: 1120~1280px
Readable Text Width: 640~760px
Public Profile Max Width: 980~1120px
Admin Max Width: 1280~1440px
```

### 7.3 여백

이 서비스는 빽빽한 정보 앱이 아니라 갤러리형 기록 서비스다. 여백을 충분히 사용한다.

- 랜딩 페이지는 섹션 간 여백을 크게 둔다.
- 카드 내부는 사진/제목/메타 정보의 호흡을 둔다.
- 정비 타임라인은 기록 간 간격이 너무 좁지 않게 한다.

## 8. 주요 UI 컴포넌트 스타일

### 8.1 Header

모바일:

- 왼쪽: 로고 또는 서비스 아이콘
- 중앙/왼쪽: 페이지 제목 또는 검색 진입
- 오른쪽: 메뉴, 로그인, 프로필
- 스크롤 시 얇은 border 또는 blur background 적용

데스크톱:

- 로고
- 주요 네비게이션
- 검색
- 로그인/프로필
- CTA 버튼

주의:

- 헤더는 너무 무겁게 만들지 않는다.
- 공개 장비 페이지에서는 장비 사진이 주인공이므로 헤더는 절제한다.

### 8.2 Mobile Menu Drawer

참고 이미지처럼 단순하고 넓은 메뉴 구성을 사용한다.

예시 메뉴:

```txt
홈
장비 둘러보기
인기 장비
정비 기록 예시
부품 리뷰
커뮤니티
공지사항
내 차고
로그인/가입
```

디자인:

- 전체 화면 또는 거의 전체 화면 오버레이
- 큰 메뉴 텍스트
- 충분한 세로 간격
- 접히는 카테고리
- 상단 닫기 버튼

주의:

- 메뉴에 너무 많은 기능을 넣지 않는다.
- MVP에서는 핵심 메뉴만 노출한다.

### 8.3 Search Bar

탐색 페이지와 랜딩 중간에 검색창을 제공한다.

검색 대상:

- 장비 모델명
- 브랜드
- 부품명
- 정비 유형
- 사용자 닉네임

스타일:

- 둥근 pill 형태
- 연한 배경
- 왼쪽 아이콘
- placeholder는 구체적으로 작성

예:

```txt
모델명, 부품명, 정비 기록 검색
```

### 8.4 Cards

카드는 서비스의 핵심 UI다.

종류:

- 장비 카드
- 정비 기록 카드
- 부품 카드
- 카테고리 카드
- 게시글 카드
- 공지 카드
- 프리미엄 스킨 카드

공통 스타일:

- 둥근 모서리
- 얇은 border
- 부드러운 shadow 또는 shadow 없음
- 큰 이미지 영역
- 명확한 제목
- 작은 메타 정보
- hover/tap feedback

주의:

- 카드가 쇼핑몰 상품 카드처럼 보이지 않게 한다.
- 장비 사진과 기록의 감성이 살아야 한다.

## 9. 랜딩 페이지 구성

랜딩 페이지는 참고 이미지처럼 섹션별로 콘텐츠를 보여주되, Maniac Garage의 서비스 가치를 명확히 전달해야 한다.

### 9.1 추천 구조

```txt
1. Header
2. Hero
3. Featured Garage / 샘플 장비 페이지
4. Maintenance Timeline Preview
5. Parts & Tuning Preview
6. Why Maniac Garage
7. Popular Categories
8. Public Profile Showcase
9. Premium Features Preview
10. FAQ
11. Final CTA
12. Footer
```

### 9.2 Hero

목표:

- 서비스 정체성을 3초 안에 이해시킨다.

문구 예시:

```txt
내 장비의 모든 기록을
가장 멋진 페이지로.
```

서브 문구:

```txt
사진, 정비 이력, 튜닝 부품, 관리 주기를 한곳에 정리하고 공유하세요.
```

CTA:

```txt
내 차고 만들기
샘플 페이지 보기
```

비주얼:

- 큰 장비 사진 카드
- 정비 타임라인 미니 카드
- 부품 리스트 미니 카드
- 배지/스탯 카드

### 9.3 Featured Garage

참고 이미지의 추천 콘텐츠 가로 스크롤 구조를 장비 카드에 적용한다.

섹션 제목:

```txt
요즘 올라온 멋진 장비들
```

카드 정보:

- 대표 사진
- 장비명
- 모델명
- 오너 닉네임
- 정비 기록 수
- 좋아요 수

### 9.4 Maintenance Timeline Preview

정비 기록의 실용성을 보여주는 섹션.

구성:

- 타임라인 카드 3~4개
- 오일 교체
- 타이어 교체
- 체인 관리
- 튜닝 부품 장착

문구:

```txt
오일을 언제 갈았는지, 어떤 부품을 달았는지 잊지 마세요.
```

### 9.5 Popular Categories

참고 이미지의 인기 카테고리 카드 구조를 활용한다.

카테고리 예시:

- 바이크
- 커스텀 PC
- 키보드
- 자전거
- 카메라

카드 스타일:

- 왼쪽 제목/설명
- 오른쪽 미니 이미지/아이콘
- 둥근 카드
- 배경은 연한 톤

## 10. 공개 장비 페이지 디자인

공개 장비 페이지는 서비스의 핵심이다.

### 10.1 목표

- 사용자가 자랑하고 싶어야 한다.
- 보는 사람이 관리 상태를 신뢰해야 한다.
- 중고 판매 시 공유 링크로 활용할 수 있어야 한다.

### 10.2 구조

```txt
1. Cover Image
2. Equipment Header
3. Owner Comment
4. Quick Stats
5. Tuning Parts Summary
6. Maintenance Timeline
7. Gallery
8. Sale/Verification Section 이후 단계
```

### 10.3 Cover Image

- 모바일에서 상단을 크게 차지한다.
- 사진 위에 과한 텍스트를 얹지 않는다.
- 하단에 장비명과 모델 정보를 자연스럽게 배치한다.

### 10.4 Quick Stats

예:

```txt
18,230km
12 records
8 parts
2021 model
```

스타일:

- 작은 데이터 카드
- 숫자는 굵게
- 단위는 작게
- 가로 스크롤 가능

### 10.5 Maintenance Timeline

- 세로 타임라인
- 날짜/주행거리/정비 유형 표시
- 기록 사진이 있으면 썸네일 노출
- 공개/비공개는 소유자에게만 표시

디자인 톤:

- 너무 회계 장부처럼 보이면 안 된다.
- 기록 카드가 잡지형/아카이브형으로 보이게 한다.

---

## 11. 탐색/커뮤니티 페이지 디자인

### 11.1 탐색 홈

참고 이미지의 리소스 탐색 페이지처럼 섹션 기반으로 구성한다.

구조:

```txt
검색 영역
추천 장비
인기 정비 기록
인기 부품 리뷰
카테고리
최신 게시글
```

### 11.2 게시판 목록

- 카테고리별 게시판 카드
- 최신 글 2~3개 미리보기
- 공지/인기/신규 배지

### 11.3 게시글 카드

정보:

- 제목
- 작성자
- 게시판명
- 댓글 수
- 좋아요 수
- 작성일
- 썸네일 이미지

스타일:

- 텍스트 중심 게시판이라도 너무 커뮤니티 앱처럼 빽빽하게 만들지 않는다.
- 이미지가 있는 게시글은 갤러리 감성을 살린다.

## 12. 어드민 디자인 관리 기준

운영자가 디자인을 쉽게 관리할 수 있어야 하지만, 서비스의 일관성이 깨지면 안 된다.

### 12.1 어드민에서 관리 가능한 것

- 랜딩 섹션 노출 여부
- 섹션 순서
- 섹션 제목/설명
- CTA 문구/링크
- 배너 이미지
- 공지 내용
- FAQ
- 카테고리 카드 문구
- 추천 콘텐츠
- 테마 프리셋 선택

### 12.2 어드민에서 관리하면 안 되는 것

- 자유 CSS 입력
- 자유 JavaScript 입력
- 전체 HTML 직접 편집
- 디자인 토큰 무제한 수정
- 보안/결제/권한 로직

### 12.3 관리 방식

- HeroSection, FeatureGridSection, CategoryCardSection 등 섹션 컴포넌트는 코드로 정의한다.
- DB에는 섹션의 type, title, body, image, cta, sortOrder, isVisible만 저장한다.
- 섹션 type별 입력 폼을 따로 제공한다.
- 변경 전 미리보기를 제공한다.
- 배포 없이 콘텐츠를 바꿀 수 있게 하되, 레이아웃 붕괴는 막는다.

## 14. 인터랙션 원칙

### 14.1 모바일 터치 중심

- 터치 영역은 최소 44px 이상
- 스와이프 가능한 가로 카드 섹션
- 하단 safe area 고려
- 메뉴/필터는 drawer 또는 bottom sheet 사용

### 14.2 움직임

애니메이션은 과하지 않게 사용한다.

사용 예:

- 카드 hover/tap scale 아주 약하게
- drawer slide
- section fade-in
- image lazy loading blur-up

피할 것:

- 과한 패럴랙스
- 로딩이 느려지는 무거운 애니메이션
- 장비 사진을 방해하는 효과

## 15. 이미지 사용 원칙

이 서비스는 사진이 핵심이다.

### 15.1 장비 사진

- 카드에서 사진 비중을 크게 둔다.
- 흐릿하거나 잘린 이미지는 피한다.
- 썸네일 비율을 통일한다.
- 공개 페이지에서는 원본 감성을 최대한 살린다.

### 15.2 기본 이미지

사용자가 사진을 올리지 않았을 때도 화면이 허전하지 않아야 한다.

기본 이미지 스타일:

- 그라파이트 배경
- 얇은 라인 아이콘
- 카테고리별 실루엣
- 미니멀한 질감

### 15.3 이미지 비율

```txt
Equipment Card: 4:3 또는 16:10
Hero Visual: 16:9 또는 21:9
Public Cover: 16:9
Timeline Thumbnail: 1:1 또는 4:3
Category Card Image: 4:3
OG Image: 1200x630
```

## 16. 접근성

- 텍스트 대비를 충분히 확보한다.
- 포인트 컬러만으로 상태를 표현하지 않는다.
- 버튼/링크는 명확한 label을 가진다.
- 이미지에는 alt 텍스트를 제공한다.
- 모바일에서 작은 아이콘만 있는 버튼은 aria-label을 둔다.
- 메뉴 drawer는 키보드 포커스와 닫기 동작을 지원한다.

## 18. 개발 시 주의사항

- 디자인 값을 컴포넌트마다 하드코딩하지 않는다.
- Tailwind 사용 시 공통 토큰과 컴포넌트 규칙을 먼저 정한다.
- 카드/섹션/버튼 스타일은 재사용 가능한 컴포넌트로 만든다.
- 랜딩 페이지 섹션은 type 기반 렌더러 구조로 만든다.
- 모바일에서 먼저 테스트한다.
- 이미지 로딩 성능을 항상 고려한다.
- Cloudflare 캐시와 이미지 최적화 전략을 같이 설계한다.
- 어드민에서 수정 가능한 디자인 요소와 코드로 고정할 요소를 명확히 분리한다.

## 19. 최종 디자인 방향

Maniac Garage는 단순한 정비 기록 웹앱이 아니라, 사용자가 자신의 장비를 멋지게 전시하고 관리 이력을 신뢰감 있게 보여주는 서비스다.

따라서 디자인은 다음 기준을 따른다.

1. 모바일에서 먼저 아름답고 쓰기 쉬워야 한다.
2. 장비 사진이 가장 돋보여야 한다.
3. 기록과 데이터는 깔끔한 카드와 타임라인으로 정리한다.
4. 전체 톤은 모던하고 미니멀하지만, 차고/장비 문화의 힙한 감성을 가진다.
5. 운영자는 어드민에서 콘텐츠와 노출을 쉽게 바꾸되, 디자인 시스템은 코드로 안정적으로 유지한다.

이 문서는 Maniac Garage의 UI/UX 작업, 디자인 시스템 구축, 랜딩 페이지 제작, 공개 장비 페이지 개발, 어드민 콘텐츠 관리 기능을 만들 때 기준 문서로 사용한다.
