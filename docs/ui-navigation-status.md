# UI Navigation Status

이 문서는 GEAR DUCK 홈 상단 로고/메뉴 구조와 사이드 네비게이션 구현 상태를 기록한다.

## 현재 홈 헤더 구조

```txt
대상 파일: src/app/page.tsx
```

- 홈 상단 로고는 중앙 정렬한다.
- 로고 이미지는 `/img/logo.png`를 사용한다.
- 현재 크기 기준은 `h-28 sm:h-32 lg:h-36`이다.
- 로고가 커진 만큼 헤더 높이는 `min-h-32 sm:min-h-36 lg:min-h-40`을 사용한다.
- 상단의 별도 `내 기어 보기` CTA 버튼은 제거했다.
- 메뉴 버튼은 헤더 오른쪽에 단독 배치한다.

## 사이드 네비게이션 구조

```txt
대상 파일: src/shared/components/navigation/MenuButton.tsx
```

- 메뉴 버튼 클릭 시 오른쪽 사이드 패널을 연다.
- 사이드 패널 폭은 모바일 기준 `min(74vw, 20rem)`으로 제한한다.
- 사이드 패널은 `document.body`에 `createPortal`로 렌더링한다.
- 이 방식은 부모 요소의 `transform`, `overflow`, stacking context 때문에 `position: fixed` 패널이 잘리는 문제를 방지하기 위한 것이다.
- 메뉴가 열리면 body scroll을 잠근다.
- 닫기 방식은 다음을 지원한다.
  - 배경 dim 영역 클릭
  - 우측 상단 닫기 버튼
  - Escape 키

## 메뉴 항목

```txt
GearDuck
- 홈
- 기어 둘러보기
- 내 기어
- 로그인 사용자: 내 정보, 관리자면 관리 모드, 로그아웃
- 비로그인 사용자: 로그인, 회원가입

기어 카테고리
- equipmentCategories 기반 카테고리 목록
```

## 최근 해결한 UI 이슈

```txt
문제:
사이드 메뉴가 모바일에서 상단/오른쪽으로 잘려 보임.

원인:
메뉴 버튼 부모 요소의 translate transform이 fixed overlay의 기준 컨텍스트에 영향을 줄 수 있었음.

1차 조치:
헤더의 -translate-y-1/2 제거 후 calc 기반 top 위치로 변경.

추가 안정화:
MenuButton 내부 overlay를 createPortal(document.body)로 렌더링하여 부모 레이아웃 영향에서 분리.
```

## 회귀 테스트 포인트

- [ ] 모바일 홈에서 로고가 중앙 정렬되어 보인다.
- [ ] 모바일 홈에서 로고가 히어로 문구와 과도하게 겹치지 않는다.
- [ ] 메뉴 버튼이 오른쪽 상단에서 잘리지 않는다.
- [ ] 메뉴 클릭 시 사이드 패널이 화면 오른쪽에서 전체 높이로 열린다.
- [ ] 사이드 패널 폭이 화면을 과도하게 덮지 않는다.
- [ ] 배경 dim 클릭으로 메뉴가 닫힌다.
- [ ] 닫기 버튼으로 메뉴가 닫힌다.
- [ ] Escape 키로 메뉴가 닫힌다.
- [ ] 메뉴가 열려 있는 동안 배경 스크롤이 잠긴다.
- [ ] 메뉴를 닫은 뒤 배경 스크롤이 정상 복구된다.
- [ ] 로그인/비로그인 상태에 따라 메뉴 항목이 정상 표시된다.
- [ ] 카테고리 링크 클릭 시 해당 explore 페이지로 이동한다.

## 주의 사항

- 홈 로고 크기를 더 키울 때는 모바일 브라우저 주소창 높이와 히어로 섹션 첫 화면 노출 영역을 같이 확인한다.
- 사이드 패널 폭을 다시 키울 때는 360px 내외 모바일 화면에서 본문 가림 정도를 확인한다.
- 사이드 네비게이션 overlay는 portal 기반이므로 `document` 접근 전에 클라이언트 mount 여부를 확인해야 한다.
